La presente página contiene la documentación del proyecto Karaoke!, un sistema capaz de reproducir canciones con letra sincronizada para que los usuario canten. Este proyecto fue realizado en setiembre, 2021, por los usuarios:

Saymon Astúa Madrigal (astuaTEC@github.com) Gerald Salazar Elizondo (GeraldSalazar@github.com) Oscar Araya Garbanzo (OscarAraya18@github.com)

Justificación de decisiones tomadas por el equipo de desarrollo:

Lenguaje de programación: JavaScript/TypeScript:

  • Los frameworks a utilizar se basan en estos lenguajes de programación.
  • La sintaxis es relativamente sencilla.
  • Existe amplia documentación sobre ambos lenguajes.
  • TypeScript añade tipado al código Javascript, haciéndolo más fácil de leer, escribir y depurar.

Frameworks/Toolkits:

NodeJS:

  • Es un entorno de ejecución para JavaScript construido con el motor de JavaScript V8 de Chrome.
  • Necesario para utilizar Express.

Express:

  • Es una infraestructura de aplicaciones web NodeJS mínima y flexible.
  • Proporciona un conjunto sólido de características para las aplicaciones web y móviles.
  • Posee variedad de métodos HTTP a su disposición, además la creación de una API sólida es rápida y sencilla.
  • Análisis del cuerpo de las peticiones HTTP de manera eficiente.
  • Enrutado y mejor organización del código.
  • Es un framework mucho más ligero y eficiente que otros de su tipo (por ejemplo .NET Framework).

GridFS:

  • Es una API proporcionada por MongoDB para el almacenamiento de archivos, especialmente audio y video.
  • Permite manejar archivos multimedia, incluso de gran tamaño.
  • Permite “descomponer” el contenido en paquetes que se pueden enviar a la aplicación, conforme se van requiriendo.
  • Esto se adapta mejor a la idea de un servicio de streaming.

Angular:

  • Framework versátil que permite crear proyectos web de manera sencilla.
  • Gran soporte de herramientas.
  • Gran cantidad de documentación.
  • Aplicaciones fáciles de mantener.

Tecnología de base de datos: MongoDB:

  • MongoDB es una base de datos no relacional orientada a documentos.
  • Los documentos se almacenan en colecciones en formato BSON, una representación binaria de JSON.
  • El sistema requiere de almacenamiento de archivos de tamaño considerable (archivos de audio).
  • Posee el paquete de GridFS, el cual provee características que facilitan la persistencia y recuperación de archivos de audio.

Estándares de programación: GNOME programming guidelines:

  • Es un estándar de programación que tiene como finalidad el facilitar la lectura y escritura de código.
  • El código se mantiene legible aún ante el crecimiento de los proyectos.
  • El estilo de programación de los miembros del equipo se adapta a este estándar.

Proveedor de servicio en la nube: AWS:

  • Amazon Web Services (AWS) es una de las plataformas en la nube más adoptadas y completas en el mundo.
  • Ofrece más de 175 servicios integrales de centros de datos a nivel global.
  • Bajos costos de implementación (gratis para nuestro equipo de desarrollo).
  • Facilidad para la implementación y configuración de máquinas virtuales.

Organización/layout de los proyectos de código:

  • El proyecto se divide en dos grandes áreas: Frontend y Backend.
  • El Frontend está formado por cinco componentes de Angular.
  • El Backend está formador por un API construido en Express (rutas y controladores).
  • La base de datos de MongoDB está hosteada en la nube.
